ADDENDUM

Game Design:

This game was created as an interactive text-based adventure, with each level presenting choices that lead to unique outcomes. The storyline is centered around escaping a haunted house, featuring various eerie scenarios and twists to keep players engaged.

HTML Structure:

The HTML layout is divided into distinct sections: start screen, game screen, and footer. This clear separation allows easy content manipulation using JavaScript.

CSS Styling:

The CSS was designed to convey a dark and spooky theme, enhancing the haunted house atmosphere. Responsive design principles were applied to ensure it looks good on various screen sizes.

JavaScript Functionality:

JavaScript manages the game’s core functions, including level rendering, player choice handling, and game state management.

A dynamic rendering system updates the display based on the current level, including text, choices, and images.

Endgame scenarios display appropriate success or failure messages, with a restart option available.

Challenges and Solutions:

Managing the game’s branching storyline complexity was challenging, but careful structuring of level data and conditional rendering helped achieve the desired outcomes.

To maintain smooth level transitions and a cohesive story, rigorous testing and debugging were essential.

Integrating dynamic content, such as the last modified date in the footer, required proficiency in DOM manipulation and JavaScript’s Date object.

Insights Gained:

This project highlighted the importance of organized code and modular design, especially for interconnected components.

I deepened my understanding of JavaScript event handling, DOM manipulation, and CSS styling.

Testing and iterative development were key for refining the player experience and resolving bugs.
